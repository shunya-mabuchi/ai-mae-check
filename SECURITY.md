# セキュリティポリシー

AIまえチェックは、ChatGPT、Claude、Geminiなどへ文章を貼り付ける前・送信する前に、個人情報や秘密情報の消し忘れに気づくためのChrome拡張です。

このリポジトリはpublicで管理しているため、脆弱性や不具合を報告するときも、貼り付け本文、実APIキー、実トークン、実顧客名、実個人情報をIssueやPRへ書き込まないでください。

## 対象バージョン

| バージョン | 状態 |
| --- | --- |
| 0.1.x | セキュリティ修正対象 |

古い開発版やローカルビルドは、再現確認の参考にはしますが、優先的な修正対象はChrome Web Storeで公開している最新版です。

## 脆弱性の報告方法

公開Issueには、脆弱性の再現に必要な秘密情報やユーザー本文を含めないでください。

まずは次のどちらかで連絡してください。

- Chrome Web Store掲載ページに表示されるデベロッパー連絡先
- GitHub Issuesで、本文や秘密情報を含めずに「セキュリティ相談」と分かる概要だけを投稿

報告時にあると助かる情報は次の通りです。

- 利用しているAIまえチェックのバージョン
- 利用している対象サイト
- ChromeのバージョンとOS
- 期待した挙動と実際の挙動
- 本文を含まない最小限の再現手順
- DevToolsやNetworkの情報を共有する場合は、本文、Cookie、トークン、URLクエリ、アカウント識別子を削除したもの

## 公開Issueに書かないでほしい情報

- 実際の貼り付け本文、送信本文、チャット本文
- メールアドレス、電話番号、住所、氏名などの実個人情報
- APIキー、OAuth token、JWT、秘密鍵、`.env` の実値
- 顧客名、案件名、契約条件、給与、採用、法務に関する実データ
- Cloudflare、GitHub、Chrome Web Store、Hugging Faceなどの認証情報

再現に文字列が必要な場合は、`taro@example.com`、`AKIAIOSFODNN7EXAMPLE`、`ghp_dummyDummyDummyDummyDummyDummy123456` のようなダミー値を使ってください。

## 対応方針

報告を受けたら、次の観点で確認します。

1. ユーザー本文を保存・送信していない設計が破れていないか
2. 検出、マスク、安全化、送信抑止の判断が想定どおりか
3. ルール配信や署名検証の失敗時に、安全側へ倒れているか
4. 拡張権限、CSP、WebLLM Worker、Cloudflare Pages Functionsに不要なリスクがないか

重大度が高い場合は、修正PR、リリースノート、Chrome Web Storeへの更新提出を優先します。

## このツールで防げないこと

AIまえチェックは、情報漏洩を完全に防ぐ製品ではありません。検出漏れや誤検出が発生する可能性があります。WebLLMによる判定も補助的な候補提示であり、最終的に送信するかどうかはユーザー自身で判断してください。
